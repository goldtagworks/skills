# blog-studio 쉽게 이해하기 — 에이전트 · 훅 · 스킬

> blog-studio는 "주제 정하기 → 글쓰기 → 품질 검수 → 자동 발행 → 마케팅"까지
> 블로그 운영 전 과정을 AI에게 맡기는 자동화 시스템입니다.
> 이 문서는 그 안에 들어있는 **에이전트 / 훅 / 스킬**을 비개발자도 이해할 수 있게 설명합니다.

---

## 한눈에 보는 3가지

| 구분 | 한 줄 정의 | 비유 |
|------|-----------|------|
| **에이전트(Agent)** | 특정 역할을 맡은 "AI 담당자" | 팀의 직원 (전략가, 작가, 검수자…) |
| **스킬(Skill)** | 담당자가 꺼내 쓰는 "작업 매뉴얼" | 업무 매뉴얼 / 체크리스트 |
| **훅(Hook)** | 특정 순간에 자동 실행되는 "안전장치" | 자동 알림·자동 검문소 |

쉽게 말해 — **에이전트(누가)** 가 **스킬(어떻게)** 을 따라 일하고, **훅(언제 자동으로)** 이 그 과정을 지켜줍니다.

---

## 1. 에이전트 — AI 담당자 6명

블로그 팀은 크게 **콘텐츠 제작팀**과 **마케팅팀** 두 조로 나뉩니다.

### 콘텐츠 제작팀 (글을 만드는 사람들)

| 에이전트 | 역할 | 트리거 예시 |
|----------|------|-------------|
| **strategy-agent** | 콘텐츠 전략가 · SEO 전문가. 어떤 주제를 쓸지, 키워드는 뭘 노릴지 정함 | "주제 정해줘", "키워드 분석", "SEO 전략" |
| **production-agent** | 콘텐츠 제작자. 리서치 → 한글 집필 → 편집 → 영문 번역까지 담당 | "글 써줘", "번역해줘", "편집해줘" |
| **quality-agent** | 품질·애드센스 검수자. 정책 위반·품질·발행 준비 최종 점검 | "검수해줘", "애드센스 확인", "발행 준비" |

### 마케팅팀 (만든 글을 퍼뜨리는 사람들)

| 에이전트 | 역할 | 트리거 예시 |
|----------|------|-------------|
| **marketing-director** | 마케팅 총괄. 발행된 글을 어느 채널(X·Reddit·HN)에 어떻게 뿌릴지 라우팅 | "이 글 어떻게 퍼뜨릴까", "마케팅 시작" |
| **content-lead** | 카피 작가. X 스레드, Reddit 포스트, 메타 디스크립션 등 "끌어들이는 카피" 작성 | "이 글 X 스레드로", "Reddit 포스트" |
| **seo-discovery-lead** | AI-SEO 전문가. Claude·ChatGPT 같은 LLM이 우리 글을 인용하게 만듦 | "LLM이 인용할까", "AI SEO" |

> 💡 핵심: 각 에이전트는 **트리거 문장**으로 자동 호출됩니다. "글 써줘"라고 하면 production-agent가, "검수해줘"라고 하면 quality-agent가 나섭니다.

## 2. 스킬 — 작업 매뉴얼 (20여 종)

스킬은 에이전트가 실제 작업할 때 따르는 구체적인 매뉴얼입니다. 용도별로 묶으면:

**🚀 시작 · 운영**
- `start` — 첫 세팅 라우터 (환경 검증, 블로그 정체성, Blogger 연결)
- `product-marketing-context` — 블로그 정체성 문서 생성 (모든 마케팅의 출발점)
- `blog-orchestrator` — 전략→제작→검수→발행 전 과정을 자동 조율하는 지휘자
- `update-from-base` — 비개발자도 git 없이 템플릿 업데이트를 안전하게 받기

**✍️ 콘텐츠 제작**
- `content-strategy` — 주제 선정, 콘텐츠 캘린더, 경쟁 분석
- `research` — 데이터·통계 수집, 팩트체크
- `blog-writing` — 본문 작성 (심층 글 1만~1.2만 자 / 짧은 글 5천~7천 자)
- `blog-editing` — 문법·톤·가독성 교정
- `blog-localization` — 한글 → 영문 번역 + 현지화

**✅ 품질 · 발행**
- `ai-pattern-check` — 발행 직전 "AI가 쓴 티" 최종 검사 (불합격 시 발행 차단)
- `quality-check` — 품질·한영 일관성·메타데이터 최종 점검
- `adsense-optimization` — 애드센스 정책 준수·수익 최적화
- `blog-publish` — Google Blogger에 자동 발행

**📈 SEO · 마케팅**
- `seo-optimization` — 키워드 리서치, 메타 태그, 스키마 마크업
- `ai-seo` — LLM이 인용하게 만드는 AI-SEO
- `copywriting` / `copy-editing` — 마케팅 카피 작성 / 편집
- `social-content` — X·Reddit·HN 플랫폼별 포맷 가이드
- `community-marketing` — Reddit·HN·Discord 커뮤니티 운영 전략

**🛠 도구 연동**
- `codex-cli` / `codex-image` — OpenAI Codex CLI로 2차 의견·이미지 생성

> 💡 핵심: 스킬은 `/스킬이름` 형태로 직접 부르거나, 에이전트가 알아서 적절한 스킬을 골라 씁니다.

---

## 3. 훅 — 자동 안전장치 3개

훅은 사람이 시키지 않아도 **정해진 순간에 자동으로** 실행됩니다.

| 훅 | 언제 실행? | 무슨 일을 하나? |
|----|-----------|----------------|
| **session-start.sh** | 세션 시작할 때 | 지난 작업 내용·진행 이력·블로그 정체성을 자동으로 불러와 "이어서 일하기" 가능하게 함 |
| **stop-reminder.sh** | AI 응답이 끝날 때 | 작업 기록(active.md)을 안 남겼으면 "기록 남겨라"고 알림 → 다음 세션이 헤매지 않게 |
| **pre-commit-check.sh** | `git commit` 직전 | API 키·OAuth 시크릿이 실수로 커밋되는 걸 차단하는 검문소 (보안) |

> 💡 핵심: 훅 덕분에 ① 세션이 끊겨도 맥락이 유지되고, ② 비밀 키가 새어나가지 않으며, ③ 작업 기록이 자동으로 관리됩니다.

---

## 전체 흐름 한 장 요약

```
[세션 시작]
   └─ session-start 훅이 지난 맥락 자동 복원
        │
   ┌────┴─────────────────────────────────────────┐
   │  blog-orchestrator (지휘자)                    │
   │                                                │
   │  ① strategy-agent   → 주제·키워드·SEO 전략     │
   │  ② production-agent → 리서치·집필·편집·번역     │
   │  ③ quality-agent    → 애드센스·품질·AI문체 검수 │
   │  ④ blog-publish     → Blogger 자동 발행         │
   │                                                │
   │  발행 후 → marketing-director가                 │
   │     content-lead(카피) · seo-discovery-lead(AI-SEO)에 분배
   └────┬─────────────────────────────────────────┘
        │
   [git commit] → pre-commit 훅이 시크릿 검문
   [응답 종료]  → stop-reminder 훅이 기록 점검
```

---

## 한 문장 정리

> **에이전트**(전략·제작·검수·마케팅 담당자)가 **스킬**(작업 매뉴얼)을 따라 블로그를 만들고,
> **훅**(맥락 복원·기록 점검·시크릿 차단)이 그 과정을 자동으로 지켜주는 — 블로그 운영 자동화 시스템.
